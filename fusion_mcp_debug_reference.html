  • What works: pre-computed XY coordinates, explicit body targeting, MCP prompts for revolves and construction planes.

Session Start Checklist

Do this every time before opening an MCP session.

⚠ Radial Math Firewall — paste at top of every MCP session
RADIAL MATH FIREWALL — READ BEFORE ANYTHING ELSE If any step in this task requires deriving, computing, or inferring: - A radius value from a description (e.g. "outer wall", "flush with surface") - An X/Y coordinate from an angle and radius - A profile position relative to a curved surface - Any trig (sin, cos, tan) to place geometry STOP. Do not attempt to derive it. Instead: Option A — Ask: "I need the pre-computed XY corners for this cut. Can you calculate them and give me raw numbers?" Option B — Say: "I need a Python measurement to confirm [specific value] before I can place this profile." Never guess a radius. Never infer 'inner boundary' or 'outer boundary' from a description. If both radii are not given as explicit numbers: stop and ask.

Before you start

  • 1
    Paste the firewall block above your task description
    Copy from this page with the button above
  • 2
    Pre-compute all XY corners for any radial cuts
    Ask regular Claude chat (see below) — not MCP
  • 3
    Know your Z values in cm, not mm
    Divide mm by 10 — Fusion works in cm internally
  • 4
    Know your target body name exactly
    Check Fusion browser panel before opening session
  • 5
    Have volumes of each body before you start
    Run diagnostic script or check fusion_measure.txt

Using regular Claude to pre-compute (safe)

You don't need to do trig yourself. Ask Claude in a regular chat before your MCP session. You decide the input numbers — Claude does the multiplication. That split is what makes it safe.

Example — paste this into regular Claude chat
I need to pre-compute corner coordinates for a Fusion 360 cut. Just do the math — do not suggest geometry or tell me where to place anything. Angle: 330 deg Inner radius: 1.905cm Outer radius: 3.1cm Slot half-width: 0.441cm (perpendicular to radial direction) Z bottom: -12.500cm Z top: -11.700cm Give me a table of the 4 corner coordinates [X, Y, Z] I can copy directly into an MCP prompt. Nothing else.
✓ Safe — you set numbers, Claude multiplies ✗ Unsafe — Claude decides the numbers
"angle=330, r=1.905, compute X and Y" "cut the inner wall of the ring at 330"
"Z bottom is -12.5cm, Z top is -11.7cm" "slot through the outer surface"
"slot is 0.441cm each side of center" "open at the bottom of the ring"
Claude multiplies. You verify the numbers look right before using. Claude guesses a radius from the description. Wrong number enters Fusion silently.
⚠ When Claude starts guessing anyway If Claude gives coordinates you didn't provide, or says anything like:
"I'll place the profile at the outer surface..."
"The inner boundary should be around..."
"Using the tube OD as reference..."

Stop the session immediately. Do not continue with those coordinates. Restart with the firewall block and provide explicit numbers.

Core Insight — How to Think About 3D Cuts

AI models (including Claude) hallucinate when reasoning about 3D rotational geometry. The fix is to never ask the AI to reason in 3D. Instead, decompose every cut into:

  • →
    A 2D rectangle defined by flat world coordinates
  • →
    A single rotation angle (degrees)
  • →
    An operation (cut / join)
ConceptHow to express it
Shape to cutRectangle with 4 explicit world XYZ corner points
Where in ZExact Z values in cm (Fusion internal units)
Radial positionr × cos(angle), r × sin(angle) — compute beforehand, never ask AI to derive
The arcRevolve N degrees around Z axis — NOT a swept path
Which bodyName it explicitly: "Target = REF_BayonetRing_V1 only"
section 2

Why Radial Math Fails — The Root Cause

⚠ Critical — read before any cut involving radius values If you ask Claude to compute a radial position (even indirectly), it will hallucinate. The failure is consistent:

1. Claude receives "cut the outer wall only, from r=19.05mm to r=30mm"
2. Claude internally tries to resolve what "outer wall" means in 3D
3. It selects a wrong reference point or confuses OD radius with ID radius
4. The rectangle is placed at a plausible but incorrect position
5. Fusion runs without error, removes 0 material, no warning

The fix: YOU compute every radius value before giving the prompt. Pass only raw numbers. Never use words like "outer surface", "inner wall", "flush with", or "from the outside in".

Failure mode 1 — partial radius constraints

The most common trigger: describing a cut that only involves ONE radius boundary while the other is implicit.

Broken: "Cut a slot through the outer wall of the ring, 12mm wide, 8mm deep from the bottom rim"

"Outer wall" forces Claude to infer an inner boundary. It picks a plausible radius — which may be numerically correct by coincidence but the reasoning is wrong, so any variation in phrasing causes a different wrong radius.

Correct: give both radii as computed numbers:
Inner radius of cut: 1.905cm (ring ID)
Outer radius of cut: 3.1cm (ring OD 3.0 + 0.1cm buffer)

Failure mode 2 — notch cut at bottom rim

Broken: "Cut an open notch at the bottom of the ring, extending upward N mm"

"Open at the bottom" is interpreted as "start below Z_bottom". Profile origin is placed at Z_bottom − N instead of Z_bottom. Cut is entirely outside the ring body, removes 0 material.

Correct: specify Z_bottom of body as one edge, Z_bottom + height as the other:
z_bottom_edge = -12.500cm (ring bottom, Z = -125mm)
z_top_edge = -11.700cm (z_bottom + 8mm = entry slot top)

Working MCP Prompt Pattern

This pattern successfully cut all 3 bayonet lock channels. Use as a template for any rotational cut.

3.1 — Pre-compute your coordinates (you do this, not Claude)

// Example for cut at 210° angle = 210° cos(210°) = -0.866 sin(210°) = -0.500 inner_radius = 2.5cm outer_radius = 3.1cm // 3.0 + 0.1 buffer z_bottom = -11.700cm z_top = -11.150cm Corner 1 (inner, bottom): [-0.866×2.5, -0.500×2.5, -11.700] = [-2.165, -1.250, -11.700] Corner 2 (outer, bottom): [-0.866×3.1, -0.500×3.1, -11.700] = [-2.684, -1.550, -11.700] Corner 3 (outer, top): [-0.866×3.1, -0.500×3.1, -11.150] = [-2.684, -1.550, -11.150] Corner 4 (inner, top): [-0.866×2.5, -0.500×2.5, -11.150] = [-2.165, -1.250, -11.150]

3.2 — The MCP prompt template

Cut a bayonet lock channel into [BODY_NAME]. Sketch: Create a profile sketch on a construction plane containing the Z axis, facing [ANGLE]°. Draw a rectangle with these exact world coordinates: [X1, Y1, Z_BOT] [X2, Y2, Z_BOT] [X2, Y2, Z_TOP] [X1, Y1, Z_TOP] Revolve cut: Revolve that profile [DEGREES]° around the Z axis. Operation = Cut. Target body = [BODY_NAME] only. Report: Volume of [BODY_NAME] before and after. If volume removed = 0, the cut did not intersect — report and stop.
0.1cm outer radius buffer is mandatory. If your profile edges land exactly flush with the body surface, Fusion runs without error but removes zero material. Always overshoot by 0.1cm on surfaces you want to cut through.
Direction: Use negative degrees for clockwise sweep when viewed from below. Positive = counter-clockwise. For a bayonet that locks CW from the handle end: use negative degrees.
section 4

Entry Slots — Specific Prompt Pattern

Entry slots differ from lock channels in one key way: they cut through the outer wall only, with NO revolve sweep. They are straight extrude cuts open at the bottom rim.

Entry slots are not revolve cuts. They are straight extrude cuts that open at the ring bottom (Z = -125mm). Inner boundary = ring ID bore (r = 1.905cm). Outer boundary penetrates through ring OD (r = 3.0cm + 0.1cm buffer = 3.1cm).

4.1 — Entry slot MCP prompt template

Cut an entry slot into REF_BayonetRing_V1 at [ANGLE]°. This is a straight extrude cut (NOT a revolve). It opens at the ring bottom face (Z = -12.500cm) and extends upward to Z = -11.700cm. Sketch on XY plane at Z = -12.500cm (ring bottom face). Draw a rectangle with these 4 corners (all at Z = -12.500cm): [X_INNER_LEFT, Y_INNER_LEFT, -12.500] [X_OUTER_LEFT, Y_OUTER_LEFT, -12.500] [X_OUTER_RIGHT, Y_OUTER_RIGHT, -12.500] [X_INNER_RIGHT, Y_INNER_RIGHT, -12.500] Extrude: Cut upward from Z = -12.500cm to Z = -11.700cm (0.800cm depth). Operation = Cut. Target body = REF_BayonetRing_V1 only. Report volume before and after. If removed = 0, stop and report.

4.2 — Pre-computed corner coordinates for 90° slot

Slot total width = 12mm (±0.441cm from center angle). Inner r = 1.905cm. Outer r = 3.1cm. Z: -12.500 to -11.700cm.

PointX (cm)Y (cm)Z (cm)
Inner left -0.4411.905-12.500 / -11.700
Inner right +0.4411.905-12.500 / -11.700
Outer left -0.4413.100-12.500 / -11.700

For 210° and 330°: rotate the ±0.441cm slot vector by the angle, add to radial position. Keep 0.1cm outer buffer. Ask regular Claude chat to compute these — give it the numbers above as the template.

section 6

Diagnostic Workflow — When Cuts Do Nothing

When a cut runs without error but removes zero material, follow in order:

  • 1
    Confirm profile position
    Print bounding box of sketch profile. Check Y and Z ranges match body's known extents.
  • 2
    Check volume before and after
    print(f"Removed: {vol_before - vol_after:.4f} cm³") — if 0.0000 the cut missed entirely
  • 3
    Remove participantBodies and check which body was cut
    Remove it and re-run. Check all body volumes to see what was actually cut — tells you if geometry is right but targeting is wrong.
  • 4
    Add buffer to profile edges
    If geometry confirmed correct but still 0 cm³ removed, profile is flush with surface. Add 0.1–0.2cm buffer.
  • 5
    Switch to MCP prompts
    If Python keeps failing on revolves or construction planes, stop. MCP handles 3D plane creation through Fusion's UI layer which is more robust than the API.

Lessons Learned & AI Failure Modes

MCP vs Python — when to use which

Use MCP prompts forUse Python scripts for
Revolve cutsReading geometry / diagnostics
Construction plane creationSimple extrude cuts (single direction)
Complex multi-body targetingVolume checks before/after
Any cut that kept failing in PythonFinding body tokens and face data
Entry slot extrude cutsBulk ops on confirmed geometry

Debugging sequence

  • 1
    Confirm body exists with correct volume/Z range
  • 2
    Confirm profile coordinates are in the right place (print bounding box)
  • 3
    Test one cut with no participantBodies — see which body it hits
  • 4
    If geometry correct but 0 cm³ removed — add buffer to profile edges
  • 5
    If Python keeps failing on planes/revolves — switch to MCP

Profile token rules

  • →
    Profile tokens are invalidated after ANY sketch operation — re-query before every extrude
  • →
    Face query API returns faces from ALL bodies — filter by face area to avoid cross-body contamination
  • →
    Body renaming does not work via MCP — use Fusion browser right-click → rename manually
  • →
    Construction planes must be offset from TOP face using negative values to reach interior Z
  • →
    Only Z-direction extrudes cut reliably — X-direction extrudes from YZ-plane return "No target body"
